IoT Assistant

Open-source application intended to make it quick and easy to manage programmable Internet of Things devices

What is IoT Assistant?

IoT Assistant is an open-source web application. It enables to manage sensors, actuators, GPS's and cameras interfaced via MQTT, HTTP or with The Things Network.

Java based

The app is based in Java Spring Boot and its ready to be run locally on any Windows, Mac, Linux computer or cheap board (e.g. a Raspberry Pi Zero W) hosting a JVM without the need to virtualize, perform complicated setups, install or use third party apps or cloud services.

Quick and easy

It should meet the needs of most users right from the start, prioritizing simplicity over flexibility while it still allow developers to add more complexity if they have particular requirements such as customizing or integrating it with third party like Aqara or Tuya devices.

Getting started

The only step needed to run IoT Assistant is to execute the .jar file provided. Execute it by double-click on it or via command line this way:

# java -jar iot-assistant-0.0.1.jar

Depending on the Java version you may need to execute this command instead:

# java --add-opens java.base/java.lang=ALL-UNNAMED --add-opens java.base/java.lang.invoke=ALL-UNNAMED -jar iot-assistant-0.0.1.jar

Once it is running you should see a login page at http://localhost:8080. Default user and password is admin iotassistant

Once logged in click on System then Settings to configure your Iot Assistant instance to get Telegram notifications and connect to your devices. If you are not sure how to fullfill the settings see this guide.

Architecture

This application was developed in the context of this trainning course mentored by Luis Fernandez Muñoz at EscuelaIt.

Development process used is RUP. The artifacts produced by the RUP activities were used to document the architecture using Philip Krutchen 4+1 View Model. UML diagrams are located here.

ESP32 code

For programable IoT devices some microcontroller attached to some transductor is usually used. In this context, the microcontroller behaviour consist in reading the transductor value via GPIO, UART, SPI... and publishing this value via some network interface, and perhaps sleeping in between to save battery.

ESP32 is a very popular microcontroller nowadays because its cheap and has wireless capabilities. I wrote libraries and Arduino projects examples to facilitate devices development. Code and documentation is provided here.
